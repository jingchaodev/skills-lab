# Career / strategy diagram pattern for HTML notes

Use this when an HTML note translates career advice, strategy principles, operating philosophy, or abstract decision frameworks into a polished reading artifact.

## Lesson

A weak first attempt for strategy notes is a generic hub-and-spoke map: several principles around a center. It may satisfy a relationship checklist, but it usually lacks a narrative mechanism.

A stronger diagram shows a **leverage flywheel**:

```text
current raw material
  → high-density arena
  → problem selection
  → ambitious form
  → research / evaluation loop
  → last-mile artifact
  → external reputation
  → higher-quality opportunities
  ↺ feedback improves next shot
```

It can also show a red failure path:

```text
work compounds only inside a local reward system → external opportunity quality stays low
```

## Rule

For career/strategy notes, do **not** draw a generic constellation of concepts unless the source itself is a constellation. Prefer a diagram that shows the mechanism by which value compounds.

Good diagram grammars:

1. **Flywheel** — raw material → selection → artifact → reputation → opportunity → better raw material.
2. **Pipeline with trap branch** — current state → decision filter → artifact/output, with a red local-optimum path.
3. **Scorecard-to-action map** — evaluation dimensions → bottleneck → one highest-leverage action.
4. **Before/after repositioning** — current identity/reward function → transformation layer → target identity/reward function.

Avoid:

- Six principle boxes arranged around a center with vague verbs.
- A diagram that restates section headings without adding causality.
- A “nice map” that does not reveal the bottleneck, trap, or compounding loop.

## Useful content for career/strategy notes

If the note is about a technical career transition, the diagram should usually include:

- current raw material: existing role, projects, domain exposure, network,
- transformation: problem selection, practice loop, evaluation loop, artifact creation,
- target positioning: the scarce capability or role narrative,
- externalization: credible people can inspect a specific artifact,
- opportunity effect: stronger relationships and higher-quality opportunities,
- red trap: work compounds only inside a local reward system.

## Visual bar

Use a rich architecture-note style where possible:

- thesis strip at the top,
- 6–9 substantive nodes,
- each node has title + payload,
- arrows have verbs,
- one focal artifact/positioning node,
- red dashed local-optimum or failure path,
- bottom strip for target opportunities / output state,
- caption that explains the mechanism in one sentence.

## Verification

After drawing:

1. Screenshot the diagram, not just DOM-inspect it.
2. Ask whether it explains a mechanism that the prose alone does not visually reveal.
3. If deleting arrows/positions loses little meaning, it is still a dressed-up list; redraw as flywheel/pipeline/trap map.
